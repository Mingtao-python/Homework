# 第 1 周
## Reflection（工程反思）
### 为什么以前项目不够专业？
以前的项目只有一个 Python 文件，没有工程结构，没有模块化，没有 README，没有版本管理，别人无法维护。

### 如果 OpenAI 维护你的项目，哪些地方一定会改？
- 他们会：
- 拆模块
- 加 tests
- 加 docs
- 加 logging
- 加配置文件
- 重写 README
- 优化目录结构
### 如果三个人一起开发，现在项目会出现哪些问题？
- 文件冲突
- 命名混乱
- 没有统一目录
- 没有模块职责
- 没有版本管理
- 没有文档
### 为什么工程质量比代码数量更重要？
工程质量决定项目能不能维护、扩展、协作。代码数量不代表价值。
---
## Explanation Questions（理解能力）
### 为什么软件工程需要模块化？
模块化让代码职责单一、可维护、可扩展、可测试。

### 为什么 README 决定项目质量？
README 是别人理解项目的入口，没有 README，项目无法交付。

### 为什么真实公司要求统一目录？
统一目录让所有工程师能快速理解结构，提高协作效率。

### 为什么 Python 能运行，却不一定是工程？
能运行只是程序；可维护、可扩展、可交付才是工程。

### 什么叫 Professional AI Project？
有结构、有文档、有模块、有测试、有版本管理的 AI 工程。
---
## AI First Thinking
### 为什么 OpenAI 不会把所有代码写在 main.py？
因为 main.py 只能做入口，不能承担所有职责，否则无法维护。

### 为什么真实 AI 公司要求统一工程目录？
统一目录让所有工程师能快速理解项目结构，减少沟通成本。

### 为什么 Git 比复制文件更适合团队开发？
Git 能记录版本、协作、回滚、Review，而复制文件会造成混乱。

### 为什么 README 是工程的一部分？
README 是项目说明书，没有它别人无法继续开发。

### 如果未来有 20 位工程师开发本项目，现在的目录还能满足吗？
不能，因为没有模块化、没有 tests、没有 docs、没有结构。
