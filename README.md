# 第20周

## 1. Rule-Based 最大优点是什么？
规则清晰、逻辑透明，容易解释和维护。

## 2. Rule-Based 最大缺点是什么？
规则难以覆盖复杂场景，扩展性差。

## 3. KNN 最大优点是什么？
算法简单、直观，无需训练即可分类。

## 4. Logistic 最大优点是什么？
输出概率结果，稳定且可解释，适合二分类任务。

## 5. 为什么不用 GPT？
GPT 适合生成内容，但不适合结构化分析，本项目更重视可解释性和控制性。

## 6. 我们的优势是什么？
系统轻量、可控、可解释，适合教学与实验场景。

## 7. 用户为什么愿意使用？
能实时反馈学习状态，帮助理解模型逻辑，提升学习效率。

## 8. 数据最容易在哪里被污染？
在标签错误、重复提交、异常输入和日志记录阶段。

## 9. 最大的安全风险是什么？
边界绕过与隐私泄露，可能导致模型误判或数据泄漏。

## 10. 下一版优先改进什么？
增强实时反馈、提升模型解释能力、强化数据安全与隐私保护。
