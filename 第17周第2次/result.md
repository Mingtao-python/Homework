# 第17周：KNN 模型分析报告（解释题）
### 更深入版本
## 1. KNN 为什么有效？
KNN 基于“相似问题 → 相似标签”的假设。文本经过 TF‑IDF 后，相似的句子在向量空间中距离更近，因此最近邻的标签能很好代表当前输入。

## 2. K=1 为什么过拟合？
K=1 只看最近的一个样本，对噪声极度敏感。训练集中任何异常点都会直接影响预测，导致模型记住“细节噪声”而不是“整体规律”。

## 3. K=15 为什么欠拟合？
K=15 会把过多邻居平均在一起，使得类别边界变得模糊。小类会被大类淹没，模型无法捕捉细节差异，导致欠拟合。

## 4. 最容易误判哪些问题？
最容易误判的是 Math 与 Coding 边界模糊的问题，例如同时包含“sum”“python”等词的句子。它们在 TF‑IDF 空间中落在边界附近，KNN 难以区分。

## 5. 错误来自数据、特征还是模型？
主要来自 **特征不足**。  
TF‑IDF 无法理解语义，只能看词频；Math 与 Coding 的词汇有重叠；数据量小导致边界不稳定。模型本身（KNN）不是主要问题。

## 6. 攻击入口在哪里？
攻击入口在 User Input → Validator 之前。  
攻击者可以输入超长文本、乱码、重复词等，使 TF‑IDF 爆炸或模型崩溃。

## 7. 如何修复？
通过加强安全模块修复：
- 限制最大长度
- 检查非 ASCII 比例
- 检查重复词
- 拒绝空输入
- 拒绝可疑 prompt
这样可确保模型只接收正常问题，避免被恶意输入破坏。
